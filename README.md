## ⌨️ ARIES CODE — The Star Feature

<p align="center">
  <img src="https://img.shields.io/badge/ARIES%20CODE-Autonomous%20Coding%20Agent-ff6600?style=for-the-badge" alt="Aries Code"/>
</p>

**Aries Code** is a full autonomous coding agent built directly into the Aries platform. It doesn't suggest code snippets — it **architects, writes, executes, debugs, and verifies entire applications** without human intervention.

### How It Works

```
You: "Build me a REST API with user auth and a React dashboard"

Aries Code:
  📋 Phase 1: PLAN      → Analyzes requirements, creates architecture
  🏗️ Phase 2: SCAFFOLD  → Generates project structure and boilerplate  
  ⚡ Phase 3: IMPLEMENT  → Writes all source code across files
  🔧 Phase 4: FIX       → Runs the app, catches errors, patches them
  🚀 Phase 5: SERVE     → Launches with hot reload, ready to use
```

### Three Ways to Use It

#### 1. CLI Mode — For Terminal Warriors
```bash
# From any terminal, anywhere
aries-code "build a REST API with SQLite and JWT auth"
aries-code "create a real-time chat app with WebSockets"
aries-code "build a markdown blog engine with syntax highlighting"
```

#### 2. Dashboard Mode — Visual Coding
Access through the web dashboard at `http://localhost:3333/aries-code`:
- Live output streaming
- File tree visualization
- Phase progress tracking
- Error logs with fix attempts
- One-click app preview

#### 3. Multi-Agent Swarm Mode — The Big Guns
For complex projects, Aries Code deploys a **5-role agent swarm**:

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐
│  ARCHITECT  │────▶│   CODER (×N) │────▶│   REVIEWER   │
│ Plans arch  │     │ Write code   │     │ Quality check│
└─────────────┘     │ in parallel  │     └──────┬───────┘
                    └──────────────┘            │
                                          ┌────▼───────┐     ┌─────────┐
                                          │   TESTER   │────▶│  FIXER  │
                                          │ Run tests  │     │ Patches │
                                          └────────────┘     └─────────┘
```

- **Architect** — Decomposes the task, creates a technical plan
- **Coders** — Multiple agents write code **in parallel** across different files
- **Reviewer** — Checks code quality, consistency, and correctness
- **Tester** — Executes the code, runs test suites
- **Fixer** — Automatically patches any errors found by the tester

Up to **20 iteration loops** — Aries Code will keep fixing until it works or exhausts attempts.

## 🛠️ Configuration

### AI Providers

Aries works with any OpenAI-compatible API. Configure through the dashboard or `config.json`:

```json
{
  "providers": {
    "ollama": { "url": "http://localhost:11434" },
    "openai": { "apiKey": "sk-..." },
    "anthropic": { "apiKey": "sk-ant-..." },
    "openrouter": { "apiKey": "sk-or-..." }

  "defaultProvider": "ollama",
```

## ❓ FAQ

**Q: Do I need to install anything besides Node.js?**  
A: No. Aries has zero npm dependencies. Just `node launcher.js`.

**Q: Can I use it without internet?**  
A: Yes! With Ollama running locally, Aries is 100% offline capable.

**Q: Is it really free?**  
A: Yes. MIT licensed. Use local models for $0/month. Cloud APIs are optional.

**Q: How does it compare to GitHub Copilot?**  
A: Copilot suggests lines. Aries Code builds entire applications autonomously.

**Q: Can I use my own OpenAI/Anthropic API key?**  
A: Absolutely. Configure any OpenAI-compatible provider in the dashboard.

**Q: Is my code/data sent anywhere?**  
A: Never. Everything runs locally. No telemetry, no analytics, no cloud processing.

**Q: What models work best?**  
A: For coding: `qwen2.5-coder:32b` or `deepseek-coder-v2`. For general: `llama3.1:70b`.
