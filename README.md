## рҹ§  What is Aries?

Aries is not another chatbot wrapper. It's an **autonomous AI platform** that runs entirely on YOUR machine.

| What you might think it is | What it actually is |
|---|---|
| A ChatGPT clone | A full **coding agent** that builds, tests, and deploys apps |

| A toy project | A **zero-dependency platform** built from pure Node.js |

### Core Principles

- **рҹ”’ Private** вҖ” Your data never leaves your machine. Ever.
- **рҹ’° Free** вҖ” Use Ollama for zero-cost local AI. No subscriptions.
- **рҹ”§ Zero Dependencies** вҖ” Built entirely on Node.js built-ins. No `node_modules/`.
- **рҹҢҗ Any Model** вҖ” Ollama, OpenAI, Anthropic, OpenRouter, Groq, Mistral, or any OpenAI-compatible API.
- **рҹӨ– Autonomous** вҖ” Give it a task, walk away. It plans, codes, tests, and fixes on its own.

## вҢЁпёҸ ARIES CODE вҖ” The Star Feature

<p align="center">
  <img src="https://img.shields.io/badge/ARIES%20CODE-Autonomous%20Coding%20Agent-ff6600?style=for-the-badge" alt="Aries Code"/>
</p>

**Aries Code** is a full autonomous coding agent built directly into the Aries platform. It doesn't suggest code snippets вҖ” it **architects, writes, executes, debugs, and verifies entire applications** without human intervention.

### How It Works

```
You: "Build me a REST API with user auth and a React dashboard"

Aries Code:
  рҹ“Ӣ Phase 1: PLAN      вҶ’ Analyzes requirements, creates architecture
  рҹҸ—пёҸ Phase 2: SCAFFOLD  вҶ’ Generates project structure and boilerplate  
  вҡЎ Phase 3: IMPLEMENT  вҶ’ Writes all source code across files
  рҹ”§ Phase 4: FIX       вҶ’ Runs the app, catches errors, patches them
  рҹҡҖ Phase 5: SERVE     вҶ’ Launches with hot reload, ready to use
```

### Three Ways to Use It

#### 1. CLI Mode вҖ” For Terminal Warriors
```bash
# From any terminal, anywhere
aries-code "build a REST API with SQLite and JWT auth"
aries-code "create a real-time chat app with WebSockets"
aries-code "build a markdown blog engine with syntax highlighting"
```

#### 2. Dashboard Mode вҖ” Visual Coding
Access through the web dashboard at `http://localhost:3333/aries-code`:
- Live output streaming
- File tree visualization
- Phase progress tracking
- Error logs with fix attempts
- One-click app preview

#### 3. Multi-Agent Swarm Mode вҖ” The Big Guns
For complex projects, Aries Code deploys a **5-role agent swarm**:


- **Architect** вҖ” Decomposes the task, creates a technical plan
- **Coders** вҖ” Multiple agents write code **in parallel** across different files
- **Reviewer** вҖ” Checks code quality, consistency, and correctness
- **Tester** вҖ” Executes the code, runs test suites
- **Fixer** вҖ” Automatically patches any errors found by the tester

Up to **20 iteration loops** вҖ” Aries Code will keep fixing until it works or exhausts attempts.

## рҹӣ пёҸ Configuration

### AI Providers

Aries works with any OpenAI-compatible API. Configure through the dashboard or `config.json`:

```json
{
  "providers": {
    "ollama": { "url": "http://localhost:11434" },
    "openai": { "apiKey": "sk-..." },
    "anthropic": { "apiKey": "sk-ant-..." },
    "openrouter": { "apiKey": "sk-or-..." }
  },
  "defaultProvider": "ollama",
  "defaultModel": "llama3.1"
}
```

## вқ“ FAQ

**Q: Do I need to install anything besides Node.js?**  
A: No. Aries has zero npm dependencies. Just `node launcher.js`.

**Q: Can I use it without internet?**  
A: Yes! With Ollama running locally, Aries is 100% offline capable.

**Q: Is it really free?**  
A: Yes. MIT licensed. Use local models for $0/month. Cloud APIs are optional.

**Q: How does it compare to GitHub Copilot?**  
A: Copilot suggests lines. Aries Code builds entire applications autonomously.

**Q: Can I use my own OpenAI/Anthropic API key?**  
A: Absolutely. Configure any OpenAI-compatible provider in the dashboard.

**Q: Is my code/data sent anywhere?**  
A: Never. Everything runs locally. No telemetry, no analytics, no cloud processing.

**Q: What models work best?**  
A: For coding: `qwen2.5-coder:32b` or `deepseek-coder-v2`. For general: `llama3.1:70b`.
